D5 "What Actually Worked" Memo
Interpretable gains from the autonomous block

What the agent discovered

The agent ran 10 controlled experiments across two model families. Three categories of modification produced real, interpretable gains: (1) expanding the weather feature set, (2) encoding seasonality correctly, and (3) switching to a nonlinear model class. Everything else — interaction terms, rolling means, lag features — produced no improvement under linear regression.

Most important discovery: The single largest gain in the experiment log came from switching model family (Exp 10), not from adding features. Holding the Exp 6 feature set fixed and replacing lm with ranger Random Forest dropped val RMSE from 0.9716 to 0.7366 — a 24% improvement from one controlled change. This result is the most trusted in the project: one variable changed, one outcome measured, clean causal account.

Was the improvement real or accidental?

The Exp 10 result is credible. It was produced by a single controlled change from a documented baseline, evaluated against the same frozen val set (2019–2020) used for every prior run. The improvement is large enough (24%) to be well outside the noise band observed in adjacent experiments (Exp 7: +0.0014, Exp 8: +0.0009). A one-time lucky run would not explain a gap of this magnitude relative to the surrounding experiment space.

What consistently failed

Four experiment categories reliably failed to improve RMSE under linear regression: city as a factor variable (Exp 4), lag features (Exp 5), interaction terms (Exp 7), and rolling means (Exp 8). This is a coherent pattern, not random noise — a linear model is structurally unable to capture the nonlinear and cross-city relationships these features encode. The features themselves are not bad. They have not yet been retested on the Random Forest baseline, which is the primary open item for Week 6.

What comes next

The current best model (Exp 10 RF, RMSE 0.7366) used only 9 features from the Exp 6 set. The next experiment block should retest Exp 4 and Exp 5 additions on top of this RF baseline as isolated one-variable additions, then begin RF hyperparameter tuning (mtry, min.node.size) as a second axis. XGBoost should be added as a model comparison once the RF feature set is finalized.
